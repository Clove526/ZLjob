# 产品需求文档 (PRD)

## 《重生之我偷看了面试官的剧本》

### 版本：v1.0 | 状态：Phase 1-2 已完成，Phase 3 进行中

---

## 一、产品概述

### 1.1 产品定位
一款融合**奇幻RPG叙事**风格的沉浸式**面试模拟游戏**。将求职面试过程包装为"冒险家公会试炼"，通过策略性回答、Boss战机制、能力卡牌收集等游戏化设计，让用户在娱乐中提升面试技巧。

### 1.2 核心价值
- **解决痛点**：传统面试准备枯燥乏味，用户缺乏练习动力和即时反馈
- **创新体验**：将面试包装为RPG闯关，赋予"重生者偷看剧本"的独特叙事身份
- **实用价值**：虽然是游戏，但所有回答内容均为真实面试话术，具备实际指导意义

### 1.3 目标用户
- 即将求职的应届生
- 准备跳槽的职场人（1-5年经验）
- RPG游戏爱好者
- 重生/穿越网文读者

## 二、世界观与叙事设计

### 2.1 世界观设定

> 你是一位意外重生回到求职路口的前世程序员。命运给了你第二次机会，还附赠了一个不可思议的金手指——**你能"隐约看到"面试官手里的评分剧本**。但剧本是残缺的，你需要用自己的智慧和策略来填补空白。
>
> 三位面试官不是普通的面试官，他们是**冒险家公会的试炼导师**。通过他们的考验，你才能获得梦寐以求的"高级冒险家（理想职位）"资格。

## 三、核心玩法系统

### 3.1 Boss战对战系统

#### 3.1.1 HP机制

**玩家HP（信心值）：**
- 初始值：50-100（取决于开局角色选择/简历匹配度）
- 影响HP的因素：
  - Excellent回答：回复5-10 HP
  - Poor回答：损失10-15 HP
  - 保守策略：HP损失减少50%
  - HP归零 → 面试失败

**Boss HP（面试官满意度）：**
- 初始值逐关递增：第1关80 → 第2关100 → 第3关120
- 影响Boss HP的因素：
  - Excellent回答：造成15-20伤害
  - Good回答：造成8-12伤害
  - OK回答：造成3-5伤害
  - Poor回答：Boss回复5 HP
  - Boss HP归零 → 过关

#### 3.1.2 胜负判定

| 条件 | 结果 |
|------|------|
| Boss HP归零 & 收集3+卡牌 | 过关 → 进入下一关 |
| Boss HP归零 & 收集<3卡牌 | 过关但警告：卡牌不足影响结局 |
| 玩家HP归零 | 试炼失败 → 触发"重生"重来 |
| 三关全部通过 | 进入结局判定 |

### 3.2 策略选择系统

#### 3.2.1 四种策略

| 策略 | 图标 | 风格 | 伤害倍率 | 风险 |
|------|------|------|---------|------|
| 保守策略 🛡️ | 稳扎稳打 | 平淡但安全 | 0.7x | 低（HP损伤-50%） |
| 进取策略 ⚔️ | 主动出击 | 有亮点 | 1.2x | 中（失败时HP损伤+20%） |
| 创新策略 💡 | 出奇制胜 | 非常规思路 | 1.5x | 高（失败时HP损伤+50%） |
| 察言观色 👁️ | 见招拆招 | 根据Boss状态调整 | 动态 | 取决于判断准确性 |

#### 3.2.2 剧本碎片提示

每次选择策略时，系统展示"剧本碎片"提示：
- 提示内容：该题目的关键评分点
- 示例："剧本提示：这道题重点考察『项目主导力』和『量化思维』"
- 提示清晰度取决于当前进度和表现：
  - 表现好 → 提示清晰
  - 表现差 → 提示模糊

### 3.3 卡牌收集系统

#### 3.3.1 五张能力卡牌

| 卡牌 | 名称 | 获得条件 | 象征 |
|------|------|---------|------|
| 🃏1 | 技术之证 | 第2关表现优异（Good+） | 专业技能 |
| 🃏2 | 沟通之证 | 任意关卡回答体现清晰逻辑 | 沟通协作 |
| 🃏3 | 逻辑之证 | 回答体现结构化思维 | 逻辑分析 |
| 🃏4 | 坚韧之证 | 压力问题中表现出色 | 抗压能力 |
| 🃏5 | 协作之证 | 回答体现团队意识 | 团队合作 |

#### 3.3.2 收集机制

- 每轮回答后，AI自动判定是否获得新卡牌
- 卡牌不重复获得
- 卡牌进度影响最终结局

#### 3.3.3 结局系统（4种以上）

| 卡牌数 | 结局等级 | 结局描述 |
|-------|---------|---------|
| 5/5 | S级 · 完美录用 | 高级职位Offer + 隐藏剧情解锁 |
| 4/5 | A级 · 成功录用 | 理想职位Offer |
| 3/5 | B级 · 有条件录用 | 职位降级或试用期延长 |
| 1-2/5 | C级 · 待定 | 进入备选名单，需补试 |
| 0/5 | D级 · 未通过 | 面试失败，触发重生重来 |

## 四、交互与反馈设计

### 4.1 交互流程

```
[开局] 选择角色 → 进入第1关
       ↓
[备战] Boss出场动画 + 剧本碎片提示
       ↓
[提问] 面试官提出问题
       ↓
[策略] 选择四种策略之一（伴随剧本碎片提示）
       ↓
[回答] 展示对应策略的回答内容（流式输出）
       ↓
[反馈] AI评分动画（Excellent/Good/OK/Poor）
       ↓
[结算] HP条动画更新 + 卡牌获得判定
       ↓
[循环] 进入下一轮提问 / 进入下一关 / 结局判定
```

### 4.2 即时反馈要求

| 阶段 | 延迟要求 | 反馈内容 |
|------|---------|---------|
| 点击策略 | <300ms | 策略确认 + 剧本碎片消失动画 |
| AI评分 | <800ms | 评分标签出现 + 特效 |
| HP更新 | 即时 | HP条平滑动画 |
| 卡牌获得 | 即时 | 卡牌翻开动画 |
| Boss切换 | <500ms | 出场动画 + HP条重置 |

### 4.3 视觉反馈元素

- **HP条**：渐变色彩（绿→黄→红），平滑过渡动画
- **评分标签**：Excellent金色闪光 / Good蓝色光芒 / OK灰色 / Poor红色颤抖
- **卡牌动画**：获得时卡片翻转，全屏闪光提示
- **策略选择**：选项卡片hover高亮，选中后缩放效果
- **Boss出场**：每个关卡开始时有独立的入场动画和台词
