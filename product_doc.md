## 一、作品简介

《重生之我偷看了面试官的剧本》是一款融合**奇幻RPG叙事**风格的**AI驱动沉浸式面试模拟游戏**。作品将求职面试过程包装为"冒险家公会试炼"，玩家扮演"重生者"，凭借"偷看面试官内心剧本"的超能力，在Boss战式的面试关卡中运用策略回答、收集能力卡牌，最终改写命运获得理想Offer。

项目采用 **React 19 + Vite 8** 前端架构与 **Node.js + Fastify** 后端服务，集成 **DeepSeek LLM** 驱动智能评分引擎，部署于 **Netlify（前端）** 与 **Railway（后端）** 云平台，现已开放线上体验：https://zljob.netlify.app

## 二、市场调研

### 2.1 目标市场

中国每年高校毕业生超过1100万，加上社招跳槽人群，求职培训市场规模超千亿。然而，传统面试准备工具存在明显痛点：

| 现有方案 | 局限性 |
|---------|--------|
| 面经题库（牛客网、LeetCode） | 枯燥刷题，缺乏真实面试的交互感和压力感 |
| 视频课程（B站、得到） | 单向输入，无法模拟真实对话场景 |
| 真人模拟面试 | 成本高（200-500元/次），难以高频练习 |
| AI面试工具（HireVue等） | 面向企业端，求职者端体验冰冷机械 |

### 2.2 竞品分析

| 竞品 | 形态 | 劣势 | 本作差异化优势 |
|------|------|------|-------------|
| 牛客网面经 | 文字题库 | 无交互、无反馈、枯燥 | 沉浸式游戏化交互+AI即时反馈 |
| 模拟面试App | 录音回答 | 无场景感、缺乏真实对话 | RPG Boss战+多面试官动态反应 |
| ChatGPT练面试 | 对话问答 | 无结构化设计、缺乏游戏激励 | 关卡设计+卡牌收集+多结局激励 |
| 企业AI面试系统 | 单向录制 | 面试者被动、焦虑感强 | 主动策略选择+剧本偷看降低信息差 |

### 2.3 市场定位

聚焦**求职者端的面试训练**细分市场，以"游戏化+AI"双轮驱动，解决求职者"缺乏练习动力"和"缺乏高质量即时反馈"两大核心痛点。

## 三、方案介绍

### 3.1 产品方案

本产品不是传统的面试题库或视频课，而是一个**完整的游戏化面试训练系统**：

- **世界观包装**: 重生者设定+冒险家公会叙事框架
- **三关Boss战**: HR导师 → 技术法师 → 总监暗影裁决，逐关递进
- **策略对战系统**: 四种策略选择（保守/进取/创新/察言观色），策略影响评分权重和HP损益
- **卡牌收集机制**: 五张能力卡牌（技术/沟通/逻辑/坚韧/协作），收集进度决定结局评级
- **多结局系统**: 从S级完美录用到D级面试失败，11种结局提升复玩性

### 3.2 交互流程

```
角色选择 → 天赋装备 → 剧情引入 → 第1关(HR) → 第2关(技术) → 第3关(总监) → 结局判定
                      ↓              ↑失败则重生         ↑
                  每关循环: 提问 → 策略选择 → AI评分 → 反馈结算 → 卡牌判定 → 下一轮/下一关
```

### 3.3 技术方案

| 架构层 | 技术选型 | 用途 |
|-------|---------|------|
| 前端框架 | React 19 + Vite 8 | 组件化UI、快速构建 |
| 样式方案 | Tailwind CSS 4 | 响应式设计、主题定制 |
| 后端框架 | Node.js + Fastify | 高性能API服务 |
| AI引擎 | DeepSeek Chat API（兼容OpenAI） | 智能评分、对话生成、场景描述 |
| 数据库 | Supabase (PostgreSQL) | 可选持久化存储 |
| 部署平台 | Netlify（前端）+ Railway（后端） | 云原生部署 |

### 4.2 核心创新点

#### 创新一：RPG叙事+求职训练跨界融合

传统面试准备工具多采用题库刷题或视频录播形式，用户参与度低、流失率高。本产品将面试包装为三关Boss战RPG，每个面试官拥有独立人格、性格特质和专属战斗台词，用户在"打怪通关"的沉浸体验中自然完成面试训练，大幅提升用户留存和练习频次。

#### 创新二：剧本偷看系统（作弊机制游戏化）

首创"剧本偷看"玩法——用户可消耗资源"偷看"面试官的真实内心想法，揭示隐藏的考察维度、雷区预警和策略推荐。这一设计将面试准备中的"换位思考"能力训练转化为可操作的博弈机制：偷看得越多，信息优势越大，但嫌疑值积累会导致面试官信任度下降，形成"信息 vs 风险"的策略深度。

#### 创新三：LLM全链路驱动动态内容

不同于传统面试模拟使用固定题库+预设答案匹配，本产品全链路由大语言模型驱动：
- **自由文本回答**：用户以自己语言回答，AI实时分析质量、策略类型和评分
- **面试官差异化反应**：基于面试官人格特质的个性化回应生成
- **动态剧情分支**：AI根据用户表现实时生成场景描述和分支叙事
- **智能评分校准**：综合回答内容、策略选择、当前状态等多维度判定

#### 创新四：Roguelite+多维度状态体系

引入RPG角色状态系统映射面试表现，形成可量化、可视化、可优化的能力反馈闭环：

| 游戏状态 | 面试映射 | 设计价值 |
|---------|---------|---------|
| 演技值 | 回答的表现力与可信度 | 鼓励真实自然表达 |
| 精神疲劳度 | 面试后期的精力衰减 | 模拟真实面试耐力 |
| 专注力 | 逻辑连贯性 | 训练结构化思维 |
| 嫌疑值 | 回答是否过于模板化 | 约束剧本偷看频率 |

轮回重生+天赋解锁+成就收集形成Roguelite循环，鼓励用户反复练习探索不同策略和结局。

## 六、AI能力及技术方案

### 6.1 使用的模型

| 模型 | 来源 | 用途 |
|------|------|------|
| **DeepSeek Chat**（deepseek-chat） | DeepSeek | 核心AI引擎：回答评分、对话生成、场景描述、分支叙事 |
| **文本生成图像API** | Trae API | 面试官头像生成、游戏场景背景图 |

### 6.2 AI能力矩阵

| AI能力 | 输入 | 输出 | 调用时机 |
|--------|------|------|---------|
| 回答评分 | 问题+用户回答+面试官档案+游戏状态 | 评分(0-100)+策略分类+好感度变化+分析评语 | 每轮回答提交后 |
| 面试官反应生成 | 问题+回答+面试官性格+好感度 | 每位面试官的个性化回应(30-60字) | 评分完成后 |
| 场景描述生成 | 回合数+氛围+面试官状态+玩家状态 | 沉浸式场景文学描述(30-80字) | 每轮开始时 |
| 剧情分支叙事 | 问题+选择+面试官反应+当前状态 | 分支剧情描述(40-100字) | 评分结算后 |
| 剧本洞察生成 | 问题+面试官数据+历史记录+玩家状态 | 内心独白+隐藏维度+雷区预警+策略推荐 | 玩家触发剧本偷看时 |
| 面试复盘报告 | 整个游戏数据（回答记录+评分+好感度） | 风格分析+弱点识别+改进建议+综合评价 | 游戏结束时 |

### 6.3 技术方案关键实现

```javascript
// LLM服务核心架构（简化）
class LLMService {
  // AI实时评分
  async analyzeAnswer(question, userAnswer, interviewers, gameState) {
    const prompt = buildAnalysisPrompt(question, userAnswer, interviewers, gameState)
    return await this.callLLM(prompt, { temperature: 0.3 }) // 低温度保证评分稳定性
  }
  
  // 面试官个性化回应
  async generateResponse(question, answer, interviewers) {
    const prompt = buildResponsePrompt(question, answer, interviewers)
    return await this.callLLM(prompt, { temperature: 0.8 }) // 高温度保证多样性
  }
  
  // 剧本洞察生成
  async generateScriptInsight(question, interviewer, playerState) {
    const prompt = buildScriptPrompt(question, interviewer, playerState)
    return await this.callLLM(prompt, { temperature: 0.85 })
  }
}
```

## 七、应用场景

### 7.1 求职备战训练（核心场景）

应届生和跳槽职场人通过游戏化方式系统性练习面试。产品内置6大类15道精选题库，覆盖自我介绍、项目经验、职业规划、压力面试等核心场景。AI实时评分和剧本提示帮助用户"边练边学"。

### 7.2 高校就业辅导

可作为高校就业指导中心的辅助教学工具。相比传统"老师讲学生听"的模式，游戏化互动能显著提升学生参与度。AI面试报告可替代部分一对一辅导的工作量。

### 7.3 企业招聘预筛选（B端扩展）

企业可将定制面试问题配置到系统中，作为初筛阶段的AI面试工具。支持自由文本回答模式，可自定义评估维度和评分标准。

### 7.4 游戏化学习产品参考

为教育培训领域的产品经理和设计师提供游戏化设计的完整参考案例，展示将枯燥技能训练转化为沉浸式游戏体验的方法论。
